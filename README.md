# Rework study

This project want to help to solve a problem in a factory. That problem is the time dedicated to finish the machine which *it should be finished*. 
For it, the quality inspectors must to create a list in <a href="https://www.notion.so">NOTION</a> with jobs needed to finish the machine.

In this list they must to write the important machine info like model, CO number, MO number, the inspector who inspected the machine and select the work TAG's.
This TAG's contains these labels contain the theme of the work to be done for after analize the rework list.

**A image example bellow:**
![Rework list example](https://github.com/morawer/rework_study/blob/master/Readme_images/rework_list.jpg?raw=true)

The program collect every rework lists in the last week and get the differents datas like the info machine, the TAG's and count the quantity of *to do's*.

That info is written like a *html* with the rework list ordered with more *to do's* to less, the TAG's appear with the percentage of their use on all machines 
and finally the graph created with python is attached with the history of *to do's* for each week and the total average.

**A graph example bellow:**
![Rework list example](https://github.com/morawer/rework_study/blob/master/Readme_images/avg_week_16_graph.png?raw=true)

Thanks to this graph we can quickly see what the trend is in rework lines in the last ten weeks.
